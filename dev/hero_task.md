## 一、技能搜索与调研方法论

### 核心文件

| 文件 | 用途 |
|------|------|
| `unit.ini` | 英雄单位ID、名称、属性、技能列表(abilList) |
| `ability.ini` | 技能基础数值：冷却(Cool)、消耗(Cost)、伤害系数(DataA-F)、工具提示(Ubertip) |
| `war3map.j` | 所有触发逻辑、实际伤害计算、专属装备判断 |
| `item.ini` | 物品名称、属性加成、ID对应关系 |

> **⚠️ 大文件警告**：war3map.j 有 42万+ 行，item.ini 极大。**禁止整文件读取**，始终先 grep 定位行号，再 Read 片段。

---

### 步骤一：找英雄单位ID

```bash
# 通过中文名 grep（编码问题可能失败，失败则用目录/上下文推断）
grep -n "Name.*英雄名" unit.ini
# 找到段落头 [XXXX]，XX 即单位类型ID
```

如果 grep 中文失败，改为在 `hero_ability_init`（war3map.j 第 **365197** 行）附近直接搜技能名：
```bash
grep -n "hero_ability_add" war3map.j | head -80
```

> **特殊情况**：部分英雄技能不走 `hero_ability_add`，而是用 `UnitAddAbility(gg_unit_XXXX_NNNN, 'ABCD')` 在初始化区域添加（如时崎狂三 Nsjs ≈ 第 395818 行）。遇到在 hero_ability_init 找不到的英雄，用以下方式搜：
```bash
grep -n "UnitAddAbility.*Nsjs" war3map.j
```

---

### 步骤二：找技能ID

**方法A（优先）**：读 hero_ability_init 中对应的 `hero_ability_add` 行：
```
hero_ability_add('单位ID', 'Q技能ID', "Q名", 'W技能ID', "W名", ...)
```

**方法B**：读 unit.ini 中该单位段落的 `abilList` 字段（但注意 abilList 包含基础能力和技能，需过滤）。

**方法C（验证）**：
```bash
grep -n "^\[A0TB\]" ability.ini
# 读 Name/DataA-F 字段确认是否与技能名匹配
```

---

### 步骤三：查技能基础数值（ability.ini）

```bash
grep -n "^\[A0TB\]" ability.ini
# 得到行号后
# Read offset=行号 limit=30
```

关键字段说明：
- `Cool` = 冷却时间（单位秒，等级1数值）
- `Cost` = 魔法消耗
- `DataA` \~ `DataF` = 各等级系数（通常只取第1个逗号前的数值 = Lv1）
- `Ubertip` = 技能工具提示文字（含伪造数值，**不可盲信**，要和触发器对照）

---

### 步骤四：找触发器实现（war3map.j）

#### 方法A：通过技能ID搜触发入口
```bash
grep -n "GetSpellAbilityId() == 'A0TB'" war3map.j
# 返回行号 → 向上找 function Trig_XXX → 读该函数内容
```

#### 方法B：新式命名英雄（直接函数名）

已知使用新式命名的英雄：

| 英雄 | 函数前缀 | 示例 |
|------|---------|------|
| 五河琴里 | `hero_qingli_` | `hero_qingli_Q_time1`, `hero_qingli_W`, `hero_qingli_E` |
| 蕾米莉亚 | `hero_leimi_` | `hero_leimi_Q`, `hero_leimi_E` |
| 暗杀者(H06C) | `tina_` | `tina_W`, `tina_E_time`, `tina_R_time` |

搜索方式：
```bash
grep -n "^function hero_qingli_" war3map.j
grep -n "^function tina_" war3map.j
```

#### 方法C：搜专属物品效果
```bash
# 触发内判断英雄是否持有某物品
grep -n "YDWEUnitHasItemOfTypeBJNull.*'I0DK'" war3map.j
# 或物品拾取事件
grep -n "GetItemTypeId.*'I0DK'" war3map.j
# 或实例绑定（英雄出生时持有）
grep -n "YDWEGetItemOfTypeFromUnitBJNull.*'I0DK'" war3map.j
```

---

### 步骤五：读触发代码

```bash
# 找到函数名后，定位函数起始行
grep -n "^function Trig_LH1" war3map.j
# 然后 Read offset=行号 limit=80~150
# 大函数需分段：每次 limit≤150，多次 Read
```

**常见伤害公式模式**：

| JASS 代码 | 含义 |
|-----------|------|
| `GetHeroStr(un, true)` | 当前力量（含加成） |
| `GetHeroAgi(un, true)` | 当前敏捷 |
| `GetHeroInt(un, true)` | 当前智力 |
| `GetHeroSAI(un, true)` | 全属性之和（STR+AGI+INT） |
| `bj_HEROSTAT_STR/AGI/INT` | 属性类型常量（用于 ModifyHeroStat 等） |
| `ATTACK_TYPE_CHAOS` | 混乱攻击（无视护甲） |
| `DAMAGE_TYPE_DEMOLITION` | 毁灭伤害 |
| `DAMAGE_TYPE_UNIVERSAL` | 通用伤害 |
| `GetRandomInt(a, b)` | 随机整数 a\~b |
| `GetRandomReal(a, b)` | 随机实数 a\~b |

---

### 注意事项

1. **工具提示 ≠ 真实数值**：Ubertip 是策划写的说明文字，实际触发代码才是真值。凡是「描述与实测差异」标注均来自对触发器的实际验证。
2. **中文 grep 失败**：遇编码问题直接改用4字符 ID 搜索，不要用中文 grep。
3. **颜色代码**：Name 字段含 `|cffRRGGBB...|r` 格式色彩标记，展示时已剥离。
4. **实例 vs 类型**：`gg_unit_Nsjs_0707` 是地图上特定实例（用于 UnitAddAbility），其类型 ID 是 `Nsjs`。
5. **触发器可能有多个入口**：同一技能的即时效果和 timer 效果在不同函数里，例如 `tina_R` 和 `tina_R_time`。

## 四、已发现的通用数值规律（跨英雄）

调研过程中发现以下模式，可供后续英雄调研参考：

### 伤害公式命名惯例
- 以力量为主的近战英雄：触发函数通常名为 `Trig_[英雄缩写][数字]Func...`
- 新版函数命名：`hero_[拼音]_Q/W/E/R` 或 `hero_[拼音]_[技能名拼音]`

### 专属物品检测模式
```jass
// 硬专属（YDWEGetItemOfType，物品直接绑定在单位身上）
YDWEGetItemOfTypeFromUnitBJNull(gg_unit_XXXX_NNNN, 'IYYY') != null

// 软专属（YDWEUnitHasItemOfType，运行时检测背包）
YDWEUnitHasItemOfTypeBJNull(GetTriggerUnit(), 'IYYY') == true
```

### 全属性之和的实现
```jass
GetHeroSAI(un, true)  // = STR + AGI + INT（含加成）
```
